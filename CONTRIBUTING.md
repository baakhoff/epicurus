# Contributing to epicurus

Thanks for your interest! The full contributor guide — workflow, commit
conventions, code style, and testing — lives in the developer documentation:

➡️ **[Developer Guide → Contributing](docs/developer/contributing.md)**

In short: open an issue, branch from `main`, make the change with tests, run the
gates (`task check`), and open a pull request that links the issue (`Closes #N`).

By contributing you agree that your contributions are licensed under the
project's [AGPL-3.0](LICENSE).
